Useful details for reports

  • Chrome or Chromium version.
  • Altreurl extension version.
  • Pattern format: wildcard or regex.
  • Source URL pattern and redirect target URL with private hosts or tokens removed.
  • Credential mode and credential source.
  • Whether the rule is waiting for credential sync, invalid, disabled, or ready.
  • Privacy-safe diagnostics or logs copied from the Diagnostics dialog.

Known limitations

  • Chrome Web Store handles automatic updates after installation. Load unpacked development builds still need manual reloads.
  • Request learning needs one matching source request before redirected requests can use captured credentials.
  • The cookie credential source cannot sync request headers. Use request learning or browser storage for headers.
  • Some protected headers cannot be copied because of Chrome extension API limits.
  • Browser storage sync requires an already-open source-origin tab.
  • The popup matches contextual rules by source domain or origin, not by full request path.
